Application Scope & Definition

What we are building, why, and the boundaries of v1.0.

What WooBarcode Pro Is

A WooCommerce plugin with two distinct but connected purposes:

Operations Tool (Admin)
Generate, design, and print barcode labels for any product or variation. Manage a print queue. Bulk import EAN codes. Export barcode images. Trigger printing from order events.
Quick Add Tool (Mobile)
A mobile-optimised page that lets a staff member scan a Price Template barcode, type a product name, take a photo, and save a new WooCommerce product — all in under 20 seconds, from a phone.

Workflow Definitions

Step-by-step flows for every major user journey. These are the ground truth for what the UI must support.

WF-1 — First-Time Setup
Install Plugin
→
DB Tables Created
→
Settings Wizard
→
Default Label Template Auto-Created
→
Plugin Ready
  1. Admin installs and activates plugin.
  2. Activation hook creates 3 DB tables: wcbp_label_templates, wcbp_price_templates, wcbp_print_queue.
  3. A default "Avery 5160" label template is inserted automatically so the plugin works out of the box.
  4. Admin is redirected to Settings → Basic tab.
  5. Admin selects: barcode symbology, SKU prefix (e.g. SBM-), auto-SKU toggle.
  6. Admin saves settings. Done. All features now usable.
WF-2 — Create a Price Template
Price Templates Page
→
New Template
→
Fill: Name, Price, Category
→
Choose Label Template
→
Save → Barcode Auto-Generated
→
Print Template Barcode
  1. Admin goes to WooCommerce → Barcode Pro → Price Templates.
  2. Clicks "Add New". Form: name, price, default category, default tags, default attributes, label template to use.
  3. On save, plugin generates a unique internal barcode value (WCBP-TPL-{id}) for this template.
  4. Admin prints this template barcode (one small label per template).
  5. Staff stick template barcodes somewhere accessible (e.g. wall chart, binder).
  6. Template is never listed as a product in the store — it is internal only.
ℹ
Price Templates are not WooCommerce products. They are rows in the wcbp_price_templates table. Scanning their barcode on the Quick Add page triggers form pre-fill — nothing else.
WF-3 — Quick Add a Product (Mobile, Core Workflow)
Open Quick Add on Phone
→
Scan Template Barcode
→
Form Pre-fills
→
Type Name
→
Tap Camera
→
Take Photo
→
Save & Next
→
Product Created + Barcode Generated
  1. Staff opens Chrome on Android → navigates to /wp-admin/?page=wcbp-quick-add (bookmarked on home screen).
  2. Full-screen simplified page loads — no WP sidebar, large touch targets, dark mode friendly.
  3. Staff scans a Price Template barcode (via Bluetooth scanner or camera scan button).
  4. AJAX call validates the barcode → returns template data (price, category, attributes).
  5. Form appears pre-filled: Price ($50), Category (Shadowboxes), Attributes (Size: Medium).
  6. Staff types the product name (e.g. "Blue Ocean Memory Box").
  7. Staff taps the camera icon → native camera opens (input capture="environment").
  8. Photo taken → JS resizes to max 1200px / 500KB → thumbnail shown inline.
  9. Staff taps "Save & Next" — product saved via WP REST API, barcode auto-generated, form clears.
  10. OR taps "Save & Print" — same as above but product also added to print queue.
  11. Total time target: under 20 seconds per product.
WF-4 — Design a Label Template
  1. Admin goes to Barcode Pro → Label Templates → Add New.
  2. Names the template (e.g. "Small Shelf Tag").
  3. Selects a preset or enters custom dimensions (width, height, cols, gap, page margin).
  4. Chooses layout: Vertical or Horizontal.
  5. Toggles content fields: Name (on), Price (on), SKU (on), Attributes (off), Logo (off), Custom Meta (off).
  6. Drags barcode-height ratio slider to 60%.
  7. Live preview panel updates instantly (no page reload) using JsBarcode + CSS.
  8. Saves template. It is now available in the Print Queue and Quick Add dropdowns.
  9. Admin can set any template as "Default" — used whenever no template is explicitly chosen.
WF-5 — Print Queue: From Add to Physical Label
Trigger
→
Item Added to Queue
→
Queue Manager Page
→
Select Label Template
→
Print Page / PDF
→
Physical Labels
  1. Items reach the queue via any trigger: Quick Add save, metabox button, bulk action, order hook.
  2. Admin/Staff opens Barcode Pro → Print Queue.
  3. Reviews items. Edits quantities inline if needed. Removes unwanted items.
  4. Selects label template from dropdown (default pre-selected).
  5. Clicks "Print" → Print Page opens in new tab. Admin chrome is hidden via print.css.
  6. Labels rendered in a grid matching the label dimensions. Sheet preview shown.
  7. User presses Ctrl+P (or clicks Print button) → browser print dialog.
  8. OR clicks "Download PDF" → server generates PDF via mPDF → file downloads.
  9. After printing, items marked as "printed" (optional, user confirms).
WF-6 — Bulk Product Import via CSV
  1. Admin prepares a CSV: columns = name, price, category_slug, ean (optional), sku (optional).
  2. Goes to Settings → Advanced → Bulk Import section.
  3. Uploads CSV file (or pastes raw CSV text).
  4. Plugin parses and shows a preview table: X valid rows, Y rows with issues (highlighted).
  5. Admin reviews, optionally skips flagged rows, then clicks "Import".
  6. Products created in batches of 50 with a progress bar (AJAX polling).
  7. Each product gets an auto-SKU if not provided. EAN stored if provided.
  8. On completion: "X products created. [Print All Barcodes] [View Products]".
WF-7 — Order-Triggered Auto Queue
  1. Admin enables "Order Auto-Queue" in Settings → Advanced. Selects trigger status (default: "Processing").
  2. Customer places an order. Order status changes to "Processing".
  3. Plugin hook fires: woocommerce_order_status_changed.
  4. For each line item: product_id (or variation_id) + ordered quantity added to print queue.
  5. If product has no barcode yet → auto-generate SKU → generate barcode → then queue.
  6. Staff sees queue grow. Prints picking/packing labels on demand.
  7. If order is later cancelled: queue items for that order are flagged/removed (configurable).
WF-8 — Scan Existing Product to Edit
  1. Staff opens Quick Add page on mobile.
  2. Scans a barcode that is already assigned to an existing product.
  3. System detects the barcode matches an existing product (checks SKU and _wcbp_ean meta).
  4. Shows: "Product found: Blue Ocean Memory Box — $50" with two options: [Edit Product] [Add to Queue].
  5. If Edit: a minimal edit form opens with name, price, stock, image fields.
  6. If Add to Queue: quantity prompt → added directly to print queue → confirmation.

Item BC-04 Needs a DecisionWhen the symbology is set to EAN-13 but the SKU is non-numeric (e.g. "SBM-123"), the plugin cannot silently produce a valid EAN-13. ★
Migration StrategyTable creation runs inside register_activation_hook using dbDelta(). Plugin version stored in wcbp_db_version option. On each update, plugins_loaded hook checks version and runs any migration needed — forward-only, never destructive.
